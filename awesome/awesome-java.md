## Projects

### Architecture

_Frameworks and libraries that help implementing and verifying design and architecture concepts._

- [ArchUnit](https://github.com/TNG/ArchUnit) [![GitHub stars](https://img.shields.io/github/stars/TNG/ArchUnit?style=flat)](https://github.com/TNG/ArchUnit/stargazers) - Test library for specifying and asserting architecture rules.
- [jMolecules](https://github.com/xmolecules/jmolecules) [![GitHub stars](https://img.shields.io/github/stars/xmolecules/jmolecules?style=flat)](https://github.com/xmolecules/jmolecules/stargazers) - Annotations and interfaces to express design and architecture concepts in code.

### Artificial Intelligence

_Frameworks that help you to leverage LLMs and AI._

- [Anahata ASI](https://github.com/anahata-os/anahata-asi) [![GitHub stars](https://img.shields.io/github/stars/anahata-os/anahata-asi?style=flat)](https://github.com/anahata-os/anahata-asi/stargazers) - Java agent container with local LLM adapters, stateful tool execution, context management and IDE integration.
- [Dokimos](https://github.com/dokimos-dev/dokimos) [![GitHub stars](https://img.shields.io/github/stars/dokimos-dev/dokimos?style=flat)](https://github.com/dokimos-dev/dokimos/stargazers) - Evaluation framework for LLM and AI-agent applications that scores responses, validates tool calls and execution traces, and catches quality regressions in CI.
- [JamJet](https://github.com/jamjet-labs/jamjet) [![GitHub stars](https://img.shields.io/github/stars/jamjet-labs/jamjet?style=flat)](https://github.com/jamjet-labs/jamjet/stargazers) - Agent runtime with a Java SDK for building AI agents, supporting graph-based workflow orchestration, multi-agent coordination, and MCP/A2A protocols.
- [LangChain4j](https://github.com/langchain4j/langchain4j) [![GitHub stars](https://img.shields.io/github/stars/langchain4j/langchain4j?style=flat)](https://github.com/langchain4j/langchain4j/stargazers) - Simplifies integration of LLMs with unified APIs and a comprehensive toolbox.
- [MCP Java SDK](https://github.com/modelcontextprotocol/java-sdk) [![GitHub stars](https://img.shields.io/github/stars/modelcontextprotocol/java-sdk?style=flat)](https://github.com/modelcontextprotocol/java-sdk/stargazers) - Enables applications to interact with AI models and tools through a standardized interface (i.e. Model Context Protocol), supporting both synchronous and asynchronous communication patterns.
- [Regulus](https://github.com/neul-labs/regulus) [![GitHub stars](https://img.shields.io/github/stars/neul-labs/regulus?style=flat)](https://github.com/neul-labs/regulus/stargazers) - Google ADK plugin suite that adds runtime compliance profiles, audit envelopes and GRC adapters for regulated Java AI agents.
- [simple-openai](https://github.com/sashirestela/simple-openai) [![GitHub stars](https://img.shields.io/github/stars/sashirestela/simple-openai?style=flat)](https://github.com/sashirestela/simple-openai/stargazers) - Library to use the OpenAI API (and compatible ones) in the simplest possible way.
- [Spring AI](https://spring.io/projects/spring-ai) - Application framework for AI engineering for Spring.

### Bean Mapping

_Frameworks that ease bean mapping._

- [dOOv](https://github.com/doov-io/doov) [![GitHub stars](https://img.shields.io/github/stars/doov-io/doov?style=flat)](https://github.com/doov-io/doov/stargazers) - Provides fluent API for typesafe domain model validation and mapping. It uses annotations, code generation and a type safe DSL to make bean validation and mapping fast and easy.
- [Immuto](https://github.com/karunarathnad/immuto) [![GitHub stars](https://img.shields.io/github/stars/karunarathnad/immuto?style=flat)](https://github.com/karunarathnad/immuto/stargazers) - Annotation processor that generates type-safe mapper implementations for Java Records using canonical constructors, with zero runtime reflection.
- [JMapper](https://github.com/jmapper-framework/jmapper-core) [![GitHub stars](https://img.shields.io/github/stars/jmapper-framework/jmapper-core?style=flat)](https://github.com/jmapper-framework/jmapper-core/stargazers) - Uses byte code manipulation for lightning-fast mapping. Supports annotations and API or XML configuration.
- [MapStruct](https://github.com/mapstruct/mapstruct) [![GitHub stars](https://img.shields.io/github/stars/mapstruct/mapstruct?style=flat)](https://github.com/mapstruct/mapstruct/stargazers) - Code generator that simplifies mappings between different bean types, based on a convention-over-configuration approach.
- [ModelMapper](https://github.com/modelmapper/modelmapper) [![GitHub stars](https://img.shields.io/github/stars/modelmapper/modelmapper?style=flat)](https://github.com/modelmapper/modelmapper/stargazers) - Intelligent object mapping library that automatically maps objects to each other.
- [Orika](https://github.com/orika-mapper/orika) [![GitHub stars](https://img.shields.io/github/stars/orika-mapper/orika?style=flat)](https://github.com/orika-mapper/orika/stargazers) - JavaBean-mapping framework that recursively copies (among other capabilities) data from one object to another.
- [reMap](https://github.com/remondis-it/remap) [![GitHub stars](https://img.shields.io/github/stars/remondis-it/remap?style=flat)](https://github.com/remondis-it/remap/stargazers) - Lambda and method handle-based mapping which requires code and not annotations if objects have different names.
- [Selma](https://github.com/xebia-france/selma) [![GitHub stars](https://img.shields.io/github/stars/xebia-france/selma?style=flat)](https://github.com/xebia-france/selma/stargazers) - Annotation processor-based bean mapper.

### Build

_Tools that handle the build cycle and dependencies of an application._

- [Apache Maven](https://maven.apache.org) - Declarative build and dependency management that favors convention over configuration. It might be preferable to Apache Ant, which uses a rather procedural approach and can be difficult to maintain.
- [Bazel](https://bazel.build) - Tool from Google that builds code quickly and reliably.
- [Buck2](https://github.com/facebook/buck2) [![GitHub stars](https://img.shields.io/github/stars/facebook/buck2?style=flat)](https://github.com/facebook/buck2/stargazers) - Encourages the creation of small, reusable modules consisting of code and resources.
- [Gradle](https://gradle.org) - Incremental builds programmed via Groovy instead of declaring XML. Works well with Maven's dependency management.

- [ReleaseRun](https://releaserun.com) - Dependency health checker for pom.xml and Gradle projects that scans for CVEs and outdated packages.
### Bytecode Manipulation

_Libraries to manipulate bytecode programmatically._

- [ASM](https://asm.ow2.io) - All-purpose, low-level bytecode manipulation and analysis.
- [Byte Buddy](https://bytebuddy.net) - Further simplifies bytecode generation with a fluent API.
- [bytecode-viewer](https://github.com/Konloch/bytecode-viewer) [![GitHub stars](https://img.shields.io/github/stars/Konloch/bytecode-viewer?style=flat)](https://github.com/Konloch/bytecode-viewer/stargazers) - Java 8 Jar & Android APK reverse engineering suite. (GPL-3.0-only)
- [Byteman](https://byteman.jboss.org) - Manipulate bytecode at runtime via DSL (rules); mainly for testing/troubleshooting. (LGPL-2.1-or-later)
- [cglib](https://github.com/cglib/cglib) [![GitHub stars](https://img.shields.io/github/stars/cglib/cglib?style=flat)](https://github.com/cglib/cglib/stargazers) - Bytecode generation library.
- [Javassist](https://github.com/jboss-javassist/javassist) [![GitHub stars](https://img.shields.io/github/stars/jboss-javassist/javassist?style=flat)](https://github.com/jboss-javassist/javassist/stargazers) - Tries to simplify bytecode editing.
- [Maker](https://github.com/cojen/maker) [![GitHub stars](https://img.shields.io/github/stars/cojen/maker?style=flat)](https://github.com/cojen/maker/stargazers) - Provides low level bytecode generation.
- [Mixin](https://github.com/SpongePowered/Mixin) [![GitHub stars](https://img.shields.io/github/stars/SpongePowered/Mixin?style=flat)](https://github.com/SpongePowered/Mixin/stargazers) - Manipulate bytecode at runtime using real Java code.
- [Perses](https://github.com/nicolasmanic/perses) [![GitHub stars](https://img.shields.io/github/stars/nicolasmanic/perses?style=flat)](https://github.com/nicolasmanic/perses/stargazers) - Dynamically injects failure/latency at the bytecode level according to principles of chaos engineering.
- [Recaf](https://www.coley.software/Recaf/) - JVM reverse engineering toolkit, essentially an IDE for Java bytecode.

### Caching

_Libraries that provide caching facilities._

- [cache2k](https://cache2k.org) - In-memory high performance caching library.
- [Caffeine](https://github.com/ben-manes/caffeine) [![GitHub stars](https://img.shields.io/github/stars/ben-manes/caffeine?style=flat)](https://github.com/ben-manes/caffeine/stargazers) - High-performance, near-optimal caching library.

- [Airline](https://rvesse.github.io/airline/) - Annotation-based framework for parsing Git-like command-line arguments.
- [JCommander](http://jcommander.org) - Command-line argument-parsing framework with custom types and validation via implementing interfaces.
- [jbock](https://github.com/jbock-java/jbock) [![GitHub stars](https://img.shields.io/github/stars/jbock-java/jbock?style=flat)](https://github.com/jbock-java/jbock/stargazers) - Reflectionless command line parser.

- [picocli](https://picocli.info) - ANSI colors and styles in usage help with annotation-based POSIX/GNU/any syntax, subcommands, strong typing for both options and positional args.

- [Jansi](https://github.com/fusesource/jansi) [![GitHub stars](https://img.shields.io/github/stars/fusesource/jansi?style=flat)](https://github.com/fusesource/jansi/stargazers) - ANSI escape codes to format console output.
- [Jexer](https://gitlab.com/AutumnMeowMeow/jexer) - Advanced console (and Swing) text user interface (TUI) library, with mouse-draggable windows, built-in terminal window manager, and sixel image support. Looks like [Turbo Vision](https://en.wikipedia.org/wiki/Turbo_Vision).

_Tools that provide metrics and quality measurements._

- [Error Prone Support](https://github.com/PicnicSupermarket/error-prone-support) [![GitHub stars](https://img.shields.io/github/stars/PicnicSupermarket/error-prone-support?style=flat)](https://github.com/PicnicSupermarket/error-prone-support/stargazers) - Error Prone extensions: extra bug checkers and a large battery of Refaster templates.

- [Spotbugs](https://github.com/spotbugs/spotbugs) [![GitHub stars](https://img.shields.io/github/stars/spotbugs/spotbugs?style=flat)](https://github.com/spotbugs/spotbugs/stargazers) - Static analysis of bytecode to find potential bugs. (LGPL-2.1-only)
- [ToolsHref](https://toolshref.com) - Online Java code analyzer and JSON-to-Mermaid visualization tool.

### Code Coverage

_Frameworks and tools that enable code coverage metrics collection for test suites._

- [Clover](https://www.atlassian.com/software/clover) - Relies on source-code instrumentation instead of bytecode instrumentation.
- [Cobertura](https://cobertura.github.io/cobertura/) - Relies on offline (or static) bytecode instrumentation and class loading to collect code coverage metrics. (GPL-2.0-only)
- [Delta Coverage](https://github.com/gw-kit/delta-coverage-plugin) [![GitHub stars](https://img.shields.io/github/stars/gw-kit/delta-coverage-plugin?style=flat)](https://github.com/gw-kit/delta-coverage-plugin/stargazers) - Computes code coverage of new and modified code based on a provided diff, supporting JaCoCo and IntelliJ coverage engines.
- [JaCoCo](https://www.eclemma.org/jacoco/) - Framework that enables collection of code coverage metrics, using both offline and runtime bytecode instrumentation.

- [CRUDGen](https://github.com/bariskokulu/CRUDGen) [![GitHub stars](https://img.shields.io/github/stars/bariskokulu/CRUDGen?style=flat)](https://github.com/bariskokulu/CRUDGen/stargazers) - Compile-time annotation processor generating CRUD layers, DTOs, JSON Patch, and custom HTTP endpoints for Spring Boot.
- [EasyEntityToDTO](https://github.com/Marcel091004/EasyEntityToDTO) [![GitHub stars](https://img.shields.io/github/stars/Marcel091004/EasyEntityToDTO?style=flat)](https://github.com/Marcel091004/EasyEntityToDTO/stargazers) - Annotation processor for automatic DTO and Mapper generation with zero boilerplate.
- [FreeBuilder](https://github.com/inferred/FreeBuilder) [![GitHub stars](https://img.shields.io/github/stars/inferred/FreeBuilder?style=flat)](https://github.com/inferred/FreeBuilder/stargazers) - Automatically generates the Builder pattern.
- [Geci](https://github.com/verhas/javageci) [![GitHub stars](https://img.shields.io/github/stars/verhas/javageci?style=flat)](https://github.com/verhas/javageci/stargazers) - Discovers files that need generated code, updates automatically and writes to the source with a convenient API.
- [Immutables](https://immutables.github.io) - Annotation processors to generate simple, safe and consistent value objects.
- [JavaPoet](https://github.com/square/javapoet) [![GitHub stars](https://img.shields.io/github/stars/square/javapoet?style=flat)](https://github.com/square/javapoet/stargazers) - API to generate source files.
- [JHipster](https://github.com/jhipster/generator-jhipster) [![GitHub stars](https://img.shields.io/github/stars/jhipster/generator-jhipster?style=flat)](https://github.com/jhipster/generator-jhipster/stargazers) - Yeoman source code generator for Spring Boot and AngularJS.
- [Joda-Beans](https://www.joda.org/joda-beans/) - Small framework that adds queryable properties to Java, enhancing JavaBeans.
- [JPA Buddy ![c]](https://www.jpa-buddy.com) - Plugin for IntelliJ IDEA. Provides visual tools for generating JPA entities, Spring Data JPA repositories, Liquibase changelogs and SQL scripts. Offers automatic Liquibase/Flyway script generation by comparing model to DB, and reverse engineering JPA entities from DB tables.
- [JSpecify Package-Info Generator](https://github.com/bcaillard/jspecify-packageinfo-generator) [![GitHub stars](https://img.shields.io/github/stars/bcaillard/jspecify-packageinfo-generator?style=flat)](https://github.com/bcaillard/jspecify-packageinfo-generator/stargazers) - Maven plugin that automatically generates package-info.java files with JSpecify annotations (@NullMarked and @NullUnmarked), helping you manage nullness boundaries in your Java projects without manual boilerplate.
